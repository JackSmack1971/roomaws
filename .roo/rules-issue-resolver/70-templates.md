# .roo/rules-issue-resolver/70-templates.md
> PR and issue templates for consistent, high-quality submissions.

## PR Template Usage
1) Save the template body to `pr-body.md`; replace placeholders.
2) Create PR with `--body-file pr-body.md`.
3) Include screenshots/video for UI changes.

## PR Title Formats (conventional)
- `fix: … (#123)` · `feat: … (#123)` · `docs:` · `refactor:` · `test:` · `chore:`

## PR Body Template Structure
- **Issue Link**: Fixes #ISSUE_NUMBER
- **Context**: Optional Roo Code task context
- **Description**: What changed and why
- **Testing**: Test procedure and verification steps
- **Screenshots**: Visual changes (when applicable)
- **Documentation**: Updates made (if any)
- **Additional Notes**: Any other relevant information
- **Discord Username**: For attribution

## Issue Template Guidelines
- Use clear, descriptive titles
- Include reproduction steps when reporting bugs
- Specify environment details (OS, browser, versions)
- Provide expected vs actual behavior
- Add relevant labels and assignees
- Include screenshots or error logs when helpful

## Template Placeholders
- `ISSUE_NUMBER`: The GitHub issue number being addressed
- `TASK_CONTEXT`: Reference to internal task or ticket
- `DESCRIPTION`: Detailed explanation of changes
- `TEST_PROCEDURE`: Steps to verify the fix/feature
- `SCREENSHOTS`: Links or embedded images
- `DOC_UPDATES`: Documentation changes made
- `ADDITIONAL_NOTES`: Extra context or considerations
- `DISCORD_USERNAME`: For community attribution
