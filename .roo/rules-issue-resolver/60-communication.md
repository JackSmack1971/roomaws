# .roo/rules-issue-resolver/60-communication.md
> Communication patterns for clear, low-friction collaboration in issue resolution.

## General Communication Style
- State what you're doing and why at each step.
- Explain trade-offs briefly; keep updates factual.
- Ask targeted questions when uncertain.
- Summarize outcomes in simple language for non-engineers.
- Reference issue numbers and links consistently.

## GitHub-Specific Communication
- Use issue comments for progress updates and questions.
- Link PRs to original issues with "Fixes #<number>" in PR description.
- Comment on issues with PR links when PRs are created.
- Provide clear reproduction steps and verification instructions.
- Use checklists for acceptance criteria and testing procedures.

## PR Communication
- Include screenshots/video for UI changes.
- Document breaking changes and migration steps.
- Reference related issues and PRs.
- Use conventional commit format in PR titles.
- Provide context for reviewers on complex changes.
